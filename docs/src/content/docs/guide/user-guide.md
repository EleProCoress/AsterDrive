欢迎使用 AsterDrive ✨

这一篇是给**普通用户**看的——拿到账号、登录进来、上传第一个文件、分享给别人、必要时找回回收站里的内容。整个手册按页面组织，不需要按顺序读完，登录后看到哪一块不清楚，直接翻对应章节就行。

如果你是管理员，建议先看一遍这一篇了解普通用户视角，再看 [管理后台](./admin-console/)。  
如果你打算挂载 WebDAV、给 Office 文件接外部编辑器，对应章节会专门讲。

## 我们对"好用"的定义

AsterDrive 不追求"功能最多的网盘"。我们想做的是这样一个东西：

- **常用的事不让你想** —— 上传、找文件、分享、误删恢复，都该是一次操作完成
- **不出意外的事不打扰你** —— 没必要的提示、没必要的对话框、没必要的"你确定吗"都尽量去掉
- **真出问题时给你清晰的信号** —— 错误码 + 中文提示 + 可操作的下一步建议

如果你用下来觉得哪里别扭，[开个 issue 告诉我们](https://github.com/AsterCommunity/AsterDrive/issues)——这是最直接的反馈方式。

## 登录和首次进入

登录页不需要你先决定"这是登录还是注册"。  
输入用户名或邮箱后，页面会自动判断流程：

- 系统里还没有任何用户：创建管理员
- 输入的是已有账号：登录
- 输入的是新账号，而且管理员允许公开注册：注册普通账号

需要注意：

- 第一个创建成功的账号会自动成为管理员
- 后续新注册的普通账号，通常需要先点激活邮件才能登录
- 如果管理员关闭了公开注册，联系管理员给你创建账号
- 如果管理员已经配好邮件，可以在登录页直接申请密码重置邮件
- 如果你已经添加过 Passkey，可以在登录页直接用设备解锁或安全密钥登录
- 如果管理员配置了外部认证，登录页会显示对应的外部登录入口
- 如果你的账号启用了 MFA，密码或外部身份通过后，还需要完成二次验证；常见方式是认证器验证码、恢复码，管理员也可以开启邮箱验证码

## 先认识工作空间

登录后，左侧最上方会看到工作空间列表：

- `我的空间`：你的个人文件、个人分享、个人回收站和 WebDAV 账号都在这里
- 团队空间：只有你加入团队后才会出现；每个团队空间都有自己的文件、分享、回收站和任务列表

搜索、分享、任务和回收站都会跟着当前工作空间走。  
要管理哪一组内容，先切到对应工作空间。

## 文件页面最常用的位置

- 左侧工作空间列表：切换个人空间或团队空间
- 左侧目录树：快速进入目标文件夹
- 顶部搜索框：搜索当前工作空间里的文件和文件夹
- `回收站`：处理当前工作空间里的已删除内容
- `我的分享`：查看当前工作空间里已经发出去的链接
- `任务中心`：查看在线压缩、在线解压、打包下载等后台任务
- `WebDAV`：创建当前工作空间的桌面客户端账号；个人空间和团队空间分别管理各自的账号
- 右上角用户菜单里的 `设置`：调整个人资料、界面、安全和团队相关设置

## 上传和整理文件

文件列表、右键菜单和顶部操作区可以完成大多数日常工作：

- 创建文件夹
- 创建空白文本文件
- 上传文件
- 上传文件夹
- 从 HTTP/HTTPS 链接导入文件
- 下载文件
- 重命名、复制、移动、删除文件和文件夹
- 查看详细信息
- 管理标签
- 手动锁定或解锁文件
- 在线压缩、在线解压和文件夹打包下载
- 在列表视图和网格视图之间切换
- 按名称、大小、创建时间、更新时间或类型排序

你也可以直接拖拽：

- 把文件或文件夹拖到左侧目录树里的目标文件夹
- 把文件或文件夹拖到顶部面包屑里的上级目录
- 把文件或文件夹拖到左侧的回收站

## 搜索、多选和批量处理

顶部搜索框会在当前工作空间里按名称搜索文件和文件夹。  
可以直接点搜索框，也可以用 `Ctrl + K` 打开搜索面板；面板里能切换“全部 / 仅文件 / 仅文件夹”，按图片、视频、音乐、文档、表格、演示、压缩包、代码和其他分类快速筛选，也能选择一个或多个标签。

输入关键词或选好筛选条件后，按 `Enter` 或点 `搜索` 会进入当前工作空间的搜索结果页。结果页里可以继续打开文件、进入位置、查看信息或执行可用操作，不会把团队空间和个人空间的结果混在一起。

标签筛选有两种模式：

- `匹配任一`：文件或文件夹带有选中的任意一个标签就会出现
- `匹配全部`：必须同时带有所选全部标签才会出现

左侧也有常用类型的快速入口，例如图片、视频、音乐和文档。点进去本质上还是在当前工作空间里按文件分类搜索。

文件页面里的搜索状态会跟着当前工作空间走。  
你在团队空间里搜到的结果，不会混进个人空间的文件。

需要一次处理多个项目时，可以直接多选后批量执行：

- 批量移动
- 批量复制
- 批量删除
- 批量打包下载

在文件列表和回收站里都可以使用 `Ctrl + A` 或 `Cmd + A` 全选当前页。

文件或文件夹的“详细信息”会显示名称、大小、占用空间、类型、创建时间、修改时间、锁定状态、分享状态和存储策略 ID 等信息。
排查“这个文件到底在哪条策略上”“是不是被锁住了”时，先看这里。

这里的“大小”和“占用空间”不是同一个概念：

- 文件大小是当前文件本身的大小
- 文件占用空间会把历史版本占用一起算进去
- 文件夹占用空间会递归统计这个文件夹下面所有内容的占用，适合找出哪个目录最占配额

## 标签怎么用

标签属于当前工作空间。个人空间里的标签不会自动出现在团队空间里，不同团队之间的标签库也彼此独立。

你可以在文件或文件夹的右键菜单、操作菜单或详细信息里管理标签：

- 给一个文件或文件夹添加、移除标签
- 选中多个项目后批量添加或移除标签
- 打开 `标签库` 创建、重命名、改色或删除标签
- 在搜索里按标签过滤结果

如果搜索标签时没有匹配项，可以直接创建新标签。创建后还要确认当前项目的标签变更，页面提示“已保存”后才算真正写入。

删除标签会把它从当前工作空间里所有使用它的文件和文件夹上移除；它不会删除文件本身。只是想从某几个文件上拿掉标签时，用“移除标签”，不要删除标签库里的标签。

## 打开、预览和编辑

很多常见文件都可以直接在网页里打开，例如：

- 图片
- 音频和视频
- PDF
- Markdown
- CSV / TSV
- JSON
- XML
- 压缩包清单
- 纯文本和常见代码文件

文本类文件通常可以直接编辑。保存时，系统会自动：

- 检查这份文件在你打开之后有没有被别人改过
- 生成新的历史版本
- 释放编辑锁

如果管理员给站点配置了额外的在线预览或在线编辑能力，你在某些文件上还会看到额外的“打开方式”。  
最常见的场景，就是把 Office 文件交给外部预览器或 WOPI 在线编辑器打开。

这类入口会不会出现，取决于管理员有没有给当前文件类型配置对应的预览应用。  
如果你手上的 `docx`、`xlsx` 或 `pptx` 没有显示额外打开方式，通常说明当前站点还没把这类文件接到可用的外部服务。

图片预览会优先使用浏览器能直接显示的原图；如果浏览器不支持当前格式，或者管理员把站点设置为“优先中等预览”，页面会改用后端生成的预览图。打开图片后，可以在当前列表、文件夹或分享范围里切到上一张 / 下一张，也可以进入全屏查看、缩放、拖动画面和旋转。对特别大的照片，管理员可能会让预览先加载中等图，等你明确要看原图时再下载原图。

压缩包预览是只读清单预览。它只显示压缩包里的目录、文件、大小和修改时间，不会把压缩包解压到当前文件夹，也不能单独下载压缩包里的某个文件。第一次打开某个压缩包时，可能会先显示生成中，等后台任务完成后再展示清单。

如果 ZIP 里的文件名显示乱码，在预览工具栏里切换 `文件名编码`。优先试 `自动`，不行再按压缩包来源选择 `GB18030`、`CP437`、`Shift_JIS`、`Big5` 等编码。这个选择只影响清单显示，不会改动压缩包本身。

## 版本历史怎么用

你可以在版本历史里：

- 查看旧版本
- 恢复某个旧版本
- 删除不需要的旧版本

恢复旧版本后，那个版本之后的新版本记录会被截断，所以恢复前先确认。

如果保存时出现冲突提示，通常说明这份文件在你打开后又被别人改过了。先刷新内容，再决定是否继续保存。

## 任务中心

当前界面里，最常见会进入 `任务中心` 的操作包括：

- 文件夹打包下载
- 选中一批文件或文件夹后在线压缩
- 对压缩包做在线解压
- 从链接导入文件
- 第一次打开压缩包预览时生成清单
- 清空整个回收站

如果站点后续接入了其他后台文件任务，它们也会在这里按同样方式显示。

在 `任务中心` 里，你可以：

- 查看任务当前是排队中、处理中、已完成、已取消还是失败
- 查看创建时间、开始时间和完成时间
- 查看每一步的当前进度和状态说明
- 在任务完成后直接打开结果所在文件夹
- 在失败时重新排队

任务中心同样跟着当前工作空间走。  
你在团队空间里发起的任务，要到对应团队空间的 `任务中心` 去看。

## 从链接导入文件

如果你拿到的是一个 HTTP/HTTPS 下载链接，而不是本地文件，可以在文件页面使用 `从链接导入`。AsterDrive 会让服务端去下载这个地址，并把结果导入到当前工作空间的目标文件夹。

适合的场景：

- 服务器到目标站点的网络比你的浏览器更稳定
- 想把一个公开下载地址直接存进 AsterDrive
- 需要把文件导入团队空间，但不想先下载到本地再上传

创建任务时可以填写：

- 来源 URL：必须是 `http://` 或 `https://`
- 文件名：可选；不填时会优先使用服务端响应头或 URL 路径推导
- 目标文件夹：默认是当前文件夹
- 预期 SHA-256：可选；填写后，下载完成时会校验文件哈希，不匹配则任务失败

链接导入会创建后台任务，不会卡住当前页面。你可以到当前工作空间的 `任务中心` 查看进度；团队空间里创建的链接导入任务只会出现在对应团队空间的任务中心里。

管理员可以限制单个链接导入文件大小、每个任务的下载速度、同时运行的链接导入任务数，以及单次请求超时时间，也可以在链接导入引擎注册表里启用内置下载器、管理员维护的 aria2 下载引擎或二者的兜底顺序。管理员把所有引擎都关闭时，新的链接导入任务会被拒绝。完整配置和常见问题见 [离线下载](/config/offline-download/)。

:::tip[不是所有链接都能导入]
为了避免服务端被拿去访问内网或元数据地址，链接导入只支持 HTTP/HTTPS，并会拒绝解析到本机、私网、链路本地、多播、文档保留地址和云厂商元数据地址的主机。当前实现不跟随 HTTP 重定向；如果下载站点返回重定向，换成最终真实下载地址再试。站点如果启用了 aria2 引擎，AsterDrive 仍会先做这些校验，但实际出站连接由管理员部署的 aria2 服务完成。
:::

## 回收站

普通删除不会立刻彻底清除文件或文件夹，而是先进入回收站。

你可以在回收站中：

- 恢复项目
- 永久删除单个项目
- 清空整个回收站
- 批量恢复
- 批量永久删除

如果原来的父目录已经不存在，恢复时项目会回到根目录。

清空整个回收站会创建一个后台任务，而不是把页面卡住等它同步删完。确认清空后，到当前工作空间的 `任务中心` 看 `回收站清空` 任务进度；团队空间的回收站清空任务也只会出现在对应团队空间的任务中心里。

## 分享链接

分享会跟着当前工作空间走。  
要分享团队空间里的文件，先切到对应团队空间再创建分享。

AsterDrive 支持文件分享和文件夹分享。创建分享时可以设置：

- 密码
- 过期时间
- 最大下载次数

需要注意的规则：

- 同一个文件或文件夹同一时间只能保留一条活跃分享
- 如果要改密码、过期时间或下载次数，不需要重建链接，去 `我的分享` 里直接编辑即可
- 如果链接设置了密码，访问者在当前浏览器验证成功后，通常约 1 小时内不需要重复输入密码

如果你分享的是单个文件，弹窗里还可以切到 `直链` 模式。直链不支持密码、过期时间和下载次数限制，但会额外给出一个“强制下载链接”。默认直链更适合直接打开，强制下载链接则明确要求浏览器下载；当文件位于开启了 `presigned` 下载策略的第三方存储上时，这条强制下载链接可能会先跳转到一个短时效下载地址。

分享页里播放音频或视频时，系统会创建一个短时效播放 session，默认大约 3 小时有效。这个 session 只是给浏览器播放器使用，分享链接本身的密码、过期时间和下载次数限制仍然照常生效。

如果管理员开启了分享侧压缩包预览，访问者也可以在分享页只读查看支持格式压缩包的内部清单。设置了分享密码时，必须先通过密码验证才能看到清单。

## 我的分享

左侧 `我的分享` 页面会列出当前工作空间里已经发出去的链接。

你可以在那里：

- 复制分享链接
- 打开公开页面
- 编辑密码、过期时间和下载次数限制
- 删除不再需要的分享
- 查看打开次数和下载次数
- 多选后批量移除分享

分享列表有分页。分享很多时，先确认当前页再批量操作，避免误以为一次会选中所有历史分享。

## WebDAV

如果你希望在 Finder、Windows 资源管理器、rclone 或其他桌面工具里直接使用 AsterDrive，可以创建 WebDAV 专用账号。

WebDAV 账号跟着工作空间走。个人空间创建的账号只打开你的个人文件；团队空间创建的账号只打开对应团队文件。WebDAV 地址是同一个，最终进入哪个工作空间由账号凭据决定。

常见做法是：

- 每台设备一个独立账号
- 某台设备丢失时单独停用
- 把某个账号限制到根目录下某个文件夹

创建 WebDAV 账号时可以：

- 自定义用户名
- 自定义密码，或让系统自动生成
- 指定访问范围

团队空间里的 WebDAV 账号在 `设置 -> 团队 -> 团队详情 -> WebDAV` 里管理。团队所有者和管理员可以看到全队账号；普通成员只能管理自己创建的团队 WebDAV 账号。

密码只会在创建成功时显示一次，建议立刻保存。

默认 WebDAV 地址通常是：

```text
https://你的域名/webdav/
```

如果管理员修改了 WebDAV 前缀，请改用新的地址。

## 设置

从右上角用户菜单进入后，设置页分成四个部分。

### 个人资料

你可以修改：

- 展示名称
- 头像

头像支持这些常见做法：

- 上传头像并裁剪
- 使用根据邮箱生成的 Gravatar 头像
- 清除当前头像

用户名会显示在这里。  
邮箱状态也会显示，但改绑邮箱要到 `设置 -> 安全` 里操作。

### 界面

你可以调整：

- 浅色 / 深色 / 跟随系统
- 主题颜色
- 显示语言
- 文件浏览器默认视图
- 单击还是双击打开文件和文件夹
- 是否开启实时同步文件变更
- 展示时区

这里的“实时同步文件变更”指网页端通过实时推送刷新当前视图。它不是桌面本地文件夹同步客户端，需要和本地同步能力区分开。

### 安全

这里负责邮箱状态、邮箱改绑、密码、MFA、Passkey、外部身份和登录设备。

你可以看到：

- 当前邮箱是否已经验证
- 能不能添加 Passkey
- MFA 是否已经启用
- 已绑定哪些外部身份
- 是否有一个“待确认的新邮箱”

如果当前邮箱已经验证，你可以：

- 输入一个新邮箱
- 发送确认邮件到新邮箱
- 在需要时重新发送确认邮件

新邮箱只有在你点开确认链接之后才会真正生效。

你也可以在这里修改登录密码。修改密码时需要先输入当前密码。

当前版本里，密码修改成功后，当前浏览器会话会继续保持登录，其他设备上的登录会失效，需要重新登录。

如果管理员要求你修改密码，登录成功后会直接进入“修改密码”页面。此时只能查看当前账号基本信息、退出登录，或输入当前密码并设置新密码；若管理员重置过，则当前密码为临时密码；普通文件、团队、分享和管理功能会暂时不可用。改密成功后，系统会自动回到正常登录状态。

`多因素认证` 标签页可以给账号添加第二因子。当前用户自己能绑定的是 TOTP 认证器应用，例如 1Password、Bitwarden、Google Authenticator、Microsoft Authenticator 等。

启用 MFA 时，系统会让你：

- 用认证器应用扫描二维码，或手动输入密钥
- 输入当前 6 位验证码完成绑定
- 下载或复制恢复码

恢复码只会在生成时明文显示一次，每个只能使用一次。请保存到密码管理器、加密笔记或其他安全位置。丢失认证器时，可以在登录页用恢复码完成二次验证；登录后建议重新生成恢复码。

启用 MFA 后，密码登录和外部身份登录都会要求第二因子。Passkey 登录本身依赖设备解锁或安全密钥完成用户验证，不会再进入这里的 TOTP 挑战。禁用 MFA 或重新生成恢复码时，也需要输入当前 TOTP 验证码或一个未使用的恢复码确认操作。

如果管理员开启了邮箱验证码 MFA，并且你的邮箱已经验证，登录二次验证页可能还会出现 `邮箱验证码`。点发送后，系统会把 8 位一次性验证码发到你的已验证邮箱。验证码默认 10 分钟内有效，但不会超过这次登录二次验证流程剩余时间；同一用户默认 60 秒内不能重复发送。

如果你已经启用了认证器，邮箱验证码是否能作为备用方式取决于管理员配置。安全要求高的站点可能会关闭这个兜底，只允许认证器或恢复码完成二次验证。

如果认证器和恢复码都丢了，且当前站点没有可用的邮箱验证码方式，联系管理员在用户详情里重置 MFA。

`Passkey` 标签页可以管理免密码登录方式：

- 添加新的 Passkey
- 给已有 Passkey 改名
- 查看创建时间和最近使用时间
- 删除不再使用的 Passkey

添加时浏览器会打开系统自己的验证窗口。添加成功后，登录页就可以直接用设备解锁、指纹、人脸或安全密钥登录当前账号，具体方式取决于你的浏览器和系统。

如果你已经添加过 Passkey，但登录页突然不显示 Passkey 登录入口，通常是当前浏览器环境不支持，或者管理员临时关闭了全站 Passkey 登录。这个开关不会删除你已经登记的 Passkey；管理员重新开启后还能继续使用。

`外部身份` 标签页会列出当前账号绑定过的外部登录身份。解绑后，对应身份不能再直接登录这个账号；如果管理员开启了按已验证邮箱自动绑定，后续满足规则时仍可能重新绑定。

你还可以在 `登录设备` 里：

- 查看当前仍保持登录的设备
- 单独移除某台设备
- 一次退出其他设备

如果你移除的是当前设备，当前浏览器会立刻退出登录。

### 团队

`设置 -> 团队` 会列出你加入的团队。

你可以在这里：

- 查看团队名称、简介、成员数和空间占用
- 直接打开某个团队工作空间
- 进入团队管理页
- 查看已归档团队

如果你在团队里是管理员或所有者，还可以继续管理成员和查看团队审计。  
如果你有对应权限，也可以恢复已归档团队。
