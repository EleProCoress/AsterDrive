这一篇按**实际场景**整理 AsterDrive 的日常动作——新部署后第一次该做什么、给 Office 文件接在线打开方式、给不同用户安排存储路线、误删后怎么找回。  
不想通读整本手册的话，直接按场景翻到对应章节照着做。

## 新部署后的第一次管理员检查

完整清单见 [首次启动检查](/deployment/runtime-behavior/)，那一篇按顺序覆盖了"自动完成了什么"和"启动后该手动检查什么"。

`常用流程` 这边只补一个：第一次登录后，按下面这个**后台入口顺序**走一遍最省事：

1. `管理 -> 概览`
2. `管理 -> 用户`
3. `管理 -> 团队`
4. `管理 -> 存储策略`
5. `管理 -> 策略组`
6. `管理 -> 任务`
7. `管理 -> 系统设置`
8. 回到个人空间后，左侧 `WebDAV`

如果你准备接从节点，再补看 `管理 -> 远程节点`。

每一项里要确认什么，[首次启动检查](/deployment/runtime-behavior/#启动后马上检查这些项) 列得更细。

## 给 Office 文件接在线打开方式

如果你准备把 `docx`、`xlsx`、`pptx` 这类文件交给外部服务处理，推荐这样做：

1. 先把 `管理 -> 系统设置 -> 站点配置 -> 公开站点地址` 填对
2. 到 `管理 -> 系统设置 -> 站点配置 -> 预览应用` 里启用或导入对应 WOPI 应用
3. 确认外部 Office / WOPI 服务能访问到 AsterDrive 的 `/api/v1/wopi/...`
4. 如果浏览器控制台明确报 AsterDrive API 的 CORS 错误，再到 `管理 -> 系统设置 -> 网络访问` 放行对应来源
5. 用一个真实 Office 文件试开并保存一次

## 上传并整理文件

整理资料时，通常按这个顺序做：

1. 先建立基础文件夹
2. 上传文件或文件夹
3. 用拖拽把它们移动到目标目录
4. 用多选批量完成移动、复制或删除

## 给不同用户或团队安排存储路线

如果你想让不同对象走不同的存储位置，推荐这样做：

1. 在 `管理 -> 存储策略` 里先创建真实的存储目标
2. 在 `管理 -> 策略组` 里定义文件大小分流规则
3. 在 `管理 -> 用户` 或 `管理 -> 团队` 里绑定对应策略组

常见做法：

- 所有人默认走本地存储
- 某些团队单独走 S3 / MinIO
- 某些团队单独走远程节点存储
- 小文件走本地，大文件走对象存储

如果路线里包含远程节点，先确认那台 follower 已经有默认远程存储目标。

没有远程存储目标时，远程存储策略本身能保存，但真正上传会被 follower 拒绝。

## 给远程节点准备远程存储目标

远程节点现在分两步：

1. `管理 -> 远程节点` 里创建节点、生成 enroll 命令，让 follower 接上主控
2. 回到这台远程节点详情，创建默认远程存储目标

第一次建议用 `local` 远程存储目标：

- 基础路径填 `default` 这类相对路径
- 勾选默认远程存储目标
- 等状态变成已应用后，再创建远程存储策略

如果 follower 最终要把对象写到 S3 / MinIO，也是在远程存储目标里新建 `s3`，不是在 enroll 命令或 Docker bootstrap ENV 里传。

## 创建团队空间

当前版本里，团队由系统管理员创建。  
推荐顺序：

1. 在 `管理 -> 团队` 里创建团队
2. 指定初始团队管理员
3. 选择这个团队要绑定的策略组
4. 让成员从左侧工作空间列表进入对应团队空间

团队创建完成后，团队里的管理员或所有者可以在 `设置 -> 团队` 里继续管理成员和查看团队审计。

## 发送资料给别人

发资料时，通常这样做：

1. 先切到正确的工作空间
2. 在文件或文件夹上创建分享链接
3. 需要保密时设置密码
4. 需要时效控制时设置过期时间
5. 需要限制传播时设置下载次数
6. 去 `我的分享` 里复制链接，并检查公开页是否正常

如果只是改密码、过期时间或下载次数，直接在 `我的分享` 里编辑原有链接即可。

## 继续未完成的上传

大文件上传中断后，常见处理方式是：

1. 回到原来的文件夹
2. 找到未完成的上传任务
3. 重新选择同一个文件

只要上传会话还没过期，系统会尽量继续未完成的部分，而不是从头开始。

## 给设备创建 WebDAV 账号

如果你要在 Finder、Windows 资源管理器、rclone 或同步工具里接入 AsterDrive，推荐这样做：

1. 先切到要连接的工作空间，再在 `WebDAV` 页面创建一个专用账号
2. 按需要限制到某个目录
3. 把 WebDAV 地址、用户名和密码填进客户端
4. 先做一次真实读写测试

这样某台设备停用时，只需要停用这一条 WebDAV 账号，不影响网页登录密码。

个人空间账号只打开个人文件；团队空间账号只打开对应团队文件。WebDAV 地址全局相同，账号凭据决定客户端进入哪个工作空间。

## 处理误删

普通删除后的项目会先进入回收站。  
最常见的处理方式是：

1. 误删后直接去回收站恢复
2. 不再需要的项目在回收站里永久删除
3. 如果要清空整个回收站，确认后去当前工作空间的 `任务中心` 看 `回收站清空` 任务进度
4. 管理员定期确认回收站保留天数是否合理

## 处理锁定问题

如果文件一直显示被锁定，常见处理顺序是：

1. 先让正在编辑它的人正常保存并退出
2. 如果是 WebDAV 客户端异常退出，再去 `管理 -> 锁` 查看
3. 确认锁已经失效后，由管理员手动清理

## 缩略图生成不符合预期

如果图片、视频或特殊格式一直没有缩略图，管理员先按这个顺序看：

1. `管理 -> 任务` 里有没有失败的缩略图任务
2. `管理 -> 系统设置 -> 文件处理 -> 媒体处理` 里对应处理器是否启用
3. 目标后缀有没有绑到正确处理器
4. 如果用了 `vips_cli` 或 `ffmpeg_cli`，测试命令是否通过
5. 源文件是否超过缩略图源文件大小上限
